RESTful API's + OAUTH/JWT
In this role I was responsible for integrating the frontend with backend services via RESTful APIs — handling things like structured API calls, 
managing response states, and ensuring smooth data flow between the UI and server. On the security side, I implemented OAuth for third-party 
authentication and JWT for session management — so tokens were issued on login, validated on protected routes, and refreshed appropriately. 
This tightened up who could access what, which we measured and saw roughly a 45 percent drop in unauthorized access incidents.

OAuth (Open Authorization) is a standard protocol for authorization. It defines a series of steps (flows) that allow an application to get limited 
access to a user's data on another service (like Google or Facebook) without ever seeing the user's password.

JWT (JSON Web Token) is a compact, URL-safe data format for securely transmitting information between parties as a JSON object. It is digitally 
signed, so you can trust the data inside hasn't been tampered with.
